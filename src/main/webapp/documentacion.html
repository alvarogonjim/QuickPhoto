API Reference

La API de QuickPhoto te permite crear, actualizar y borrar usuarios. Está diseñado como un servicio Rest. Esta página describe la API para peticiones HTTP.

La API está alojada en /api

Peticiones
HTTP URI Descripción
GET /user Devuelve todos los usuarios de la aplicación
GET /user/{userId} Devuelve el usuario con id=userId. Si el usuario no existe devuelve un “404 Not found”
POST /user Añade un nuevo usuario cuyos datos se pasan en el cuerpo de una petición en formato JSON (no se debe pasar el id, este se genera automáticamente). Si alguno de los campos no es válido (null o vacío) devuelve un error. Si se añade satisfactoriamente devuelve “201 Created” con la referencia a la URI y el contenido del usuario
PUT /user Actualiza el usuario cuyos datos se pasan en el cuerpo de la petición en formato JSON (deben incluir el id del usuario). Si el usuario no existe devuelve un “404 Not found”. Si se realiza correctamente devuelve un “200 Ok”
DELETE /user/{userId} Elimina el usuario con id = userId. Si el usuario no existe devuelve un “404 Not found”. Si se realiza correctamente devuelve un “200 Ok”
Representación Cada usuario tiene un identificador, nombre, apellidos y email. La representación del objeto JSON del recurso es:

{
  "id": "S0",
  "name": "Alvaro",
  "surName": "Gonzalez",
  "email": "alvarogonjim@gmail.com"
}
